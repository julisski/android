Local storage is just operations on a persisted table. A Room @Query that returns Flow<List<Note>> re-runs and re-emits on every write, so the UI re-renders automatically. Edit the DAO calls on the left — or tap a note on the phone — and watch the table, the SQL, and the list update live. (A teaching model, not a real database.)

Scope: this page teaches the Room part of Android storage only. If you are deciding between Room, DataStore, files, cache, MediaStore, or SAF, start with the storage chooser first.
