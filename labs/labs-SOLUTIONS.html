Lay it out

The task. Stack three lines of text vertically, evenly spaced and centered — using a Column, Arrangement, and Alignment.

Solution

Column(
    modifier = Modifier.fillMaxWidth(),
    verticalArrangement = Arrangement.spacedBy(8.dp),
    horizontalAlignment = Alignment.CenterHorizontally
) {
    Text("One")
    Text("Two")
    Text("Three")
}

How it works

  1. Make the root a Column.
  2. Put three Text composables inside it.
  3. Give it verticalArrangement = Arrangement.spacedBy(8.dp) so the lines have an even 8 dp gap.
  4. Make it Modifier.fillMaxWidth() and center the children with horizontalAlignment = Alignment.CenterHorizontally — you need width to center within.
Going deeper
  • A Column takes its children in a trailing { } block.
  • Arrangement controls the MAIN axis (vertical for a Column); Alignment controls the CROSS axis (horizontal).
  • Add modifier = Modifier.fillMaxWidth() so centering is visible across the whole width.

What the auto-checks verify

  • The root composable is a Column
  • It has exactly three direct Text children
  • 8 dp spacing via Arrangement.spacedBy(8.dp)
  • Modifier.fillMaxWidth() gives room to center within
  • Children centered with CenterHorizontally

Order matters

The task. A Modifier chain is ordered. Right now the blue fills the whole box and the text sits inside it. Make the blue HUG the text instead — by reordering two modifiers.

Solution

Box(modifier = Modifier.padding(16.dp).background(Color(0xFF2563EB))) {
    Text("Hug me", color = Color.White)
}

How it works

  1. Look at the box: the chain is .background(…).padding(16.dp), so blue is painted first and the padding pushes the text inward — blue shows all around.
  2. Swap the order to .padding(16.dp).background(…).
  3. Now the 16 dp padding is a transparent margin OUTSIDE the blue, so the blue wraps tightly around the text.
Going deeper
  • Each modifier wraps everything AFTER it in the chain; the FIRST modifier is the outermost layer, the LAST sits closest to the content.
  • You only need to reorder .padding(...) and .background(...) — nothing else changes.

What the auto-checks verify

  • A Box contains a Text
  • The chain still has both .padding and .background
  • .padding(…) comes before .background(…)

Style the text

The task. Three plain lines need styling. Make a title use a type-scale role, make the second line bold and theme-colored, and make the caption italic.

Solution

Column(verticalArrangement = Arrangement.spacedBy(6.dp)) {
    Text("Saturn", style = MaterialTheme.typography.titleLarge)
    Text("Ringed gas giant", fontWeight = FontWeight.Bold, color = MaterialTheme.colorScheme.primary)
    Text("Second largest planet", fontStyle = FontStyle.Italic)
}

How it works

  1. Give the first Text a style = MaterialTheme.typography.titleLarge.
  2. Make the second Text fontWeight = FontWeight.Bold AND color = MaterialTheme.colorScheme.primary.
  3. Make the third Text fontStyle = FontStyle.Italic.
Going deeper
  • Style and color are named arguments on Text, e.g. Text("Hi", fontWeight = FontWeight.Bold).
  • Pull sizes from the type scale (titleLarge, bodyMedium…) and colors from MaterialTheme.colorScheme instead of hardcoding.

What the auto-checks verify

  • A Text is FontWeight.Bold with color = …colorScheme.primary

Build a profile card

The task. Compose a real list-item layout: a Card with a circular avatar, a name/status column that eats the leftover width, and a trailing heart icon.

Solution

Card(modifier = Modifier.fillMaxWidth()) {
    Row(
        modifier = Modifier.padding(16.dp),
        verticalAlignment = Alignment.CenterVertically,
        horizontalArrangement = Arrangement.spacedBy(12.dp)
    ) {
        Box(modifier = Modifier.size(48.dp).clip(CircleShape).background(MaterialTheme.colorScheme.primary))
        Column(modifier = Modifier.weight(1f)) {
            Text("Ada Lovelace", style = MaterialTheme.typography.titleMedium)
            Text("Online now", color = MaterialTheme.colorScheme.primary)
        }
        Icon(Icons.Default.Favorite, contentDescription = null, tint = Color.Red)
    }
}

How it works

  1. Inside the Row, add a circular avatar: a Box with Modifier.size(48.dp).clip(CircleShape).background(MaterialTheme.colorScheme.primary).
  2. Give the name Column Modifier.weight(1f) so it takes the remaining width and pushes the icon to the end.
  3. Add a trailing Icon(Icons.Default.Favorite, contentDescription = null, tint = Color.Red) after the column.
Going deeper
  • A round avatar is just a sized Box clipped to CircleShape with a background.
  • weight(1f) only works inside a Row/Column — it splits the leftover space.
  • Order inside the Row is: avatar, then the weighted Column, then the Icon.

What the auto-checks verify

  • A Card wraps a Row
  • A circular avatar — a sized Box with clip(CircleShape)
  • The name Column uses Modifier.weight(1f)
  • A trailing Icon is present

State that survives rotation

The task. This tap counter is broken: a plain var isn’t Compose state, so taps don’t recompose and the count is lost on rotation. Give it state that updates the UI and survives a configuration change — with rememberSaveable.

Solution

Column(
    modifier = Modifier.fillMaxWidth(),
    verticalArrangement = Arrangement.spacedBy(12.dp),
    horizontalAlignment = Alignment.CenterHorizontally
) {
    var count by rememberSaveable { mutableStateOf(0) }
    Text("Taps: $count", style = MaterialTheme.typography.headlineSmall)
    Button(onClick = { count++ }) { Text("Tap me") }
}

How it works

  1. Hold the count in mutableStateOf(0) — that makes it observable, so writing it triggers recomposition (the UI redraws).
  2. Wrap that in rememberSaveable { … } so the value survives recomposition AND rotation / process death (plain remember would be lost on rotation).
  3. Declare it with by so you read & write count like a normal Int: var count by rememberSaveable { mutableStateOf(0) }.
  4. Keep the button’s onClick = { count++ } so tapping still increments it.
Going deeper
  • This is the exact pattern behind selectedTab in the NavBottomTabs project — the same line keeps the chosen tab across rotation.
  • Three layers: mutableStateOf = observable; remember = survives recompose; rememberSaveable = ALSO survives rotation/process death.
  • The by delegate (plus the getValue/setValue imports) lets you skip .value and use count directly.

What the auto-checks verify

  • The count lives in mutableStateOf(…) — observable, so taps recompose
  • It’s wrapped in rememberSaveable { … } — survives rotation & process death
  • Declared with by so you use count like a plain Int
  • The button still increments it (count++)

Row: mind the other axis

The task. A Column stacks children top-to-bottom; a Row lays them left-to-right. The two knobs swap meaning: in a Row the MAIN axis is horizontal (so spacing is horizontalArrangement) and the CROSS axis is vertical (so centering is verticalAlignment). Make the star and the label share one line — vertically centered, with an 8.dp gap.

Solution

Row(
    verticalAlignment = Alignment.CenterVertically,
    horizontalArrangement = Arrangement.spacedBy(8.dp)
) {
    Icon(Icons.Default.Star, contentDescription = null, tint = Color(0xFFFFC107))
    Text("4.8 — Highly rated", style = MaterialTheme.typography.titleMedium)
}

How it works

  1. Right now the star Icon (22 dp tall) and the label sit on different baselines, glued together. They are children of a Row, so they already run left-to-right.
  2. Center them on the cross axis (vertical) with verticalAlignment = Alignment.CenterVertically — this is the Row twin of a Column’s horizontalAlignment.
  3. Add an 8 dp gap on the main axis (horizontal) with horizontalArrangement = Arrangement.spacedBy(8.dp).
Going deeper
  • Mnemonic: arrangement = the axis the Row flows along (horizontal); alignment = the axis across it (vertical). In a Column it’s the mirror image.
  • Both go in the Row(…) parameter list, not on a child: Row(verticalAlignment = …, horizontalArrangement = …) { … }.
  • Arrangement.spacedBy(8.dp) inserts the gap between children only — no leading or trailing space.

What the auto-checks verify

  • A Row holds an Icon and a Text
  • Children are centered on the cross axis — verticalAlignment = Alignment.CenterVertically
  • An 8 dp gap via horizontalArrangement = Arrangement.spacedBy(8.dp)

Spread them out

The task. Arrangement decides how leftover space on the MAIN axis is shared out. These three action buttons are bunched at the start because the Row is only as wide as its content. Make the Row full-width and spread the buttons edge-to-edge with Arrangement.SpaceBetween.

Solution

Row(
    modifier = Modifier.fillMaxWidth(),
    horizontalArrangement = Arrangement.SpaceBetween
) {
    TextButton(onClick = {}) { Text("Cancel") }
    TextButton(onClick = {}) { Text("Save draft") }
    Button(onClick = {}) { Text("Publish") }
}

How it works

  1. A Row hugs its children by default, so there is no leftover space to distribute. First add Modifier.fillMaxWidth() so the Row spans the screen.
  2. Now set horizontalArrangement = Arrangement.SpaceBetween — first child pinned to the start, last to the end, equal gaps between.
  3. Try swapping in SpaceEvenly or SpaceAround to feel the difference, then settle on SpaceBetween for an action bar.
Going deeper
  • Arrangement only has visible space to work with once the Row is wider than its children — that’s why fillMaxWidth() comes first.
  • SpaceBetween = no space at the ends; SpaceAround = half-size gaps at the ends; SpaceEvenly = equal gaps everywhere, ends included.
  • Both the modifier and the arrangement live in the Row(…) header.

What the auto-checks verify

  • A Row holds three buttons
  • The Row is full-width — Modifier.fillMaxWidth()
  • Buttons spread edge-to-edge — horizontalArrangement = Arrangement.SpaceBetween

Push it to the edge

The task. This is the most common Row pattern in real apps: a label on the left, a control on the right, full width. The trick is Modifier.weight(1f) — give it to the label and it greedily eats all the leftover space, shoving the Switch to the far end. No fixed widths, no Spacer needed.

Solution

Row(
    modifier = Modifier.fillMaxWidth().padding(16.dp),
    verticalAlignment = Alignment.CenterVertically
) {
    Text(
        "Wi-Fi",
        modifier = Modifier.weight(1f),
        style = MaterialTheme.typography.titleMedium
    )
    Switch(checked = true)
}

How it works

  1. The Row is already fillMaxWidth(), but the label and Switch hug each other at the start because neither claims the empty space.
  2. Add Modifier.weight(1f) to the label Text. weight means “take a share of the leftover space” — with one weighted child, it takes all of it.
  3. The label now stretches across the gap, so the Switch is pushed to the trailing edge. (This only works inside a Row or Column.)
Going deeper
  • Put the modifier on the child, not the Row: Text("Wi-Fi", modifier = Modifier.weight(1f), …).
  • Why not Arrangement.SpaceBetween? It works for two items, but weight scales to any number of trailing items and lets you cap or pad the label independently — it’s the more flexible tool.
  • A weighted child reserves space after the unweighted ones are measured — that’s why the fixed-size Switch keeps its size and the Text flexes.

What the auto-checks verify

  • A full-width Row holds a Text and a Switch
  • The label Text carries Modifier.weight(1f)
  • The weighted label comes before the Switch (so it pushes it to the end)

Split the row 1 : 2

The task. Lab 8 used one weight to push. Use TWO weights to divide. When several children are weighted, they split the leftover space in proportion to their weights. Give the panels weights of 1 and 2 so the right panel ends up twice as wide as the left.

Solution

Row(
    modifier = Modifier.fillMaxWidth(),
    horizontalArrangement = Arrangement.spacedBy(12.dp)
) {
    Box(modifier = Modifier.weight(1f).height(80.dp).background(MaterialTheme.colorScheme.primaryContainer)) {
        Text("1 part", modifier = Modifier.padding(12.dp))
    }
    Box(modifier = Modifier.weight(2f).height(80.dp).background(MaterialTheme.colorScheme.secondaryContainer)) {
        Text("2 parts", modifier = Modifier.padding(12.dp))
    }
}

How it works

  1. Right now neither panel is weighted, so each is only as wide as its text — leaving a big empty gap on the right.
  2. Give the left Box Modifier.weight(1f) and the right Box Modifier.weight(2f).
  3. Weights are a ratio: 1 + 2 = 3 shares, so the panels become 1/3 and 2/3 of the row’s width — they grow and shrink together as the screen resizes.
Going deeper
  • Chain weight with the other modifiers, e.g. Modifier.weight(1f).height(80.dp).background(…). Order of weight vs paint modifiers doesn’t change the share.
  • Equal weights (1f and 1f) would split 50/50; 1f and 2f gives one-third / two-thirds.
  • Because both panels are weighted, the spacedBy(12.dp) gap is subtracted first, then the remaining width is split 1 : 2.

What the auto-checks verify

  • A full-width Row holds two Box panels
  • The left panel has Modifier.weight(1f)
  • The right panel has Modifier.weight(2f) — twice the left

Stack & align

The task. Column and Row place children side by side; a Box LAYERS them on top of each other — later children draw on top. That is how you build avatars-with-badges, banners over images, FAB overlays. Here: pin a green “online” dot to the bottom-end corner of an avatar with Modifier.align.

Solution

Box {
    Box(modifier = Modifier.size(64.dp).clip(CircleShape).background(MaterialTheme.colorScheme.primary))
    Box(
        modifier = Modifier
            .align(Alignment.BottomEnd)
            .size(16.dp)
            .clip(CircleShape)
            .background(Color(0xFF00C853))
    )
}

How it works

  1. Inside the outer Box, the avatar and the status dot both sit in the top-start corner — stacked, because a Box overlaps its children.
  2. Move the dot with Modifier.align(Alignment.BottomEnd) on the dot’s own Box. align is a Box-only modifier that positions that one child within the Box.
  3. Result: a 64 dp avatar with a 16 dp dot tucked into its bottom-right corner — the classic presence indicator.
Going deeper
  • Alignment inside a Box is two-dimensional: TopStart, Center, BottomEnd, etc. Put it as the first link: Modifier.align(Alignment.BottomEnd).size(16.dp)….
  • To align every child the same way you’d set Box(contentAlignment = …) on the parent; to move just one child, use Modifier.align(…) on that child — that’s what we want here.
  • A round shape is a sized Box .clip(CircleShape) with a .background(…) — same recipe as the avatar in Lab 4.

What the auto-checks verify

  • An outer Box stacks the avatar and the dot (3 Boxes total)
  • Both inner Boxes are circular — sized + clip(CircleShape)
  • The dot is pinned bottom-end — a Box with Modifier.align(Alignment.BottomEnd)

Drill down to a planet

The task. Navigate from the categories list all the way to Mars’s detail. Pushing a key navigates forward; the back stack should end up three deep.

Solution

val backStack = rememberNavBackStack(

Going deeper
  • Each tap calls backStack.add(…) — that IS "navigate forward".
  • Mars is itemId = 4 and lives in categoryId = 1 (Rocky Planets).
  • Stuck? Edit the list on the left to add the two keys by hand.

Seed a deep-link back stack

The task. A deep link drops the user straight onto a detail screen. Hand-seed the back stack so opening navdemo://planet/5 lands on Jupiter — and Back still walks up Detail → Items → Categories.

Solution

val backStack = rememberNavBackStack(
    CategoriesKey,
    ItemsKey(categoryId = 2),
    DetailKey(itemId = 5),
)

How it works

  1. The link navdemo://planet/5 targets Jupiter (itemId = 5, in categoryId = 2).
  2. Seed the back stack with three keys so Back is meaningful: CategoriesKey, then ItemsKey(categoryId = 2), then DetailKey(itemId = 5).
  3. Add them to the rememberNavBackStack(…) list. Then press ‹ Back in the phone to confirm it walks up the chain.
Going deeper
  • A deep link doesn’t just show one screen — you build the whole stack the user would have if they had navigated there, so Back behaves.
  • Jupiter is a Gas Giant: categoryId = 2, itemId = 5.

What the auto-checks verify
