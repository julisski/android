The “Screen frame” controls rewrite the code on the left — pick the root container (Column / Box / LazyColumn / Scaffold), then change padding / alignment / system bars and watch the code update and the preview re-render. The editor’s Copy grabs the whole emulator-ready screen.

root = which container holds your screen: Column stacks vertically · Box overlaps/centers one thing · LazyColumn scrolls a long list · Scaffold is the standard app screen (gives you innerPadding). The preview simulates the status & navigation bars; with system bars = none content slides under them (the corner bug), systemBarsPadding() / Scaffold push it clear.
