WebSocket Live, hands-on

WebSocket Live teaches a second network protocol: a full-duplex wss:// connection that stays open so either side can push messages at any time — the contrast to HTTP's one-shot request/response. This page lets you drive a live socket in your browser: connect, send text, watch the echo come back, see each connection event fold into UI state through a pure reducer, and compare the protocol with HTTP.

WebSocketClient.kt — LiveConnection + Real (OkHttp) + Fake + switch
interface LiveConnection {
    fun events(): Flow<ConnectionEvent>
    fun connect(); fun send(text: String); fun close()
}

class RealWebSocketClient(
    private val url: String = "wss://echo.websocket.events",
    private val httpClient: OkHttpClient = OkHttpClient(),     // injectable for testing
) : LiveConnection {
    private var webSocket: WebSocket? = null
    private val events = MutableSharedFlow<ConnectionEvent>(extraBufferCapacity = 64)
    override fun events() = events.asSharedFlow()

    override fun connect() {
        events.tryEmit(ConnectionEvent.Connecting)
        webSocket = httpClient.newWebSocket(Request.Builder().url(url).build(),
            object : WebSocketListener() {
                override fun onOpen(webSocket: WebSocket, r: Response) { events.tryEmit(ConnectionEvent.Open) }
                override fun onMessage(webSocket: WebSocket, text: String) { events.tryEmit(ConnectionEvent.MessageReceived(text)) }
                override fun onClosing(webSocket: WebSocket, code: Int, reason: String) {
                    webSocket.close(1000, null)
                    events.tryEmit(ConnectionEvent.Closed(reason.ifBlank { "closed" }))
                }
                override fun onFailure(webSocket: WebSocket, t: Throwable, r: Response?) { events.tryEmit(ConnectionEvent.Failure(t.message ?: "failed")) }
            })
    }
    override fun send(text: String) { webSocket?.send(text) }
    override fun close() { webSocket?.close(1000, "client closed"); webSocket = null }
}

fun provideLiveConnection(useFake: Boolean = true): LiveConnection =
    if (useFake) FakeWebSocketClient() else RealWebSocketClient()

LiveViewModel.kt — state + pure reducer + collector
data class LiveUiState(
    val status: ConnectionStatus = ConnectionStatus.Disconnected,
    val messages: List<ChatMessage> = emptyList(),
)

fun reduceLiveState(state: LiveUiState, event: ConnectionEvent): LiveUiState = when (event) {
    is ConnectionEvent.Connecting      -> state.copy(status = ConnectionStatus.Connecting)
    is ConnectionEvent.Open            -> state.copy(status = ConnectionStatus.Connected)
    is ConnectionEvent.MessageReceived -> state.copy(messages = state.messages + ChatMessage(event.text, fromMe = false))
    is ConnectionEvent.Closed          -> state.copy(status = ConnectionStatus.Disconnected)
    is ConnectionEvent.Failure         -> state.copy(status = ConnectionStatus.Failed)
}

class LiveViewModel(private val connection: LiveConnection = FakeWebSocketClient()) : ViewModel() {
    private val _uiState = MutableStateFlow(LiveUiState())
    val uiState: StateFlow<LiveUiState> = _uiState.asStateFlow()
    init { viewModelScope.launch { connection.events().collect { e -> _uiState.update { reduceLiveState(it, e) } } } }
    fun connect() = connection.connect()
    fun send(text: String) {
        if (text.isBlank()) return
        connection.send(text)
        _uiState.update { it.copy(messages = it.messages + ChatMessage(text, fromMe = true)) }   // optimistic
    }
}

MainActivity.kt — open once, render the stream
@Composable
fun LiveScreen(modifier: Modifier = Modifier, viewModel: LiveViewModel = viewModel()) {
    val uiState by viewModel.uiState.collectAsStateWithLifecycle()
    LaunchedEffect(Unit) { viewModel.connect() }          // open the socket exactly once
    var input by remember { mutableStateOf("") }
    LiveContent(uiState, input, onInputChange = { input = it },
        onSend = { viewModel.send(input); input = "" }, modifier)
}

Key idea · why a socket

WebSocket vs HTTP

HTTP is one request → one response: to get three answers you open three round-trips. A WebSocket opens once and then carries any number of frames either way. Press the button and watch the difference.

HTTP (request/response)
Each message = a new connection + handshake.
WebSocket (one open connection)
One handshake, then frames flow both ways.

Key idea · one-line offline switch

Real vs Fake — the same interface, two implementations

Both implement LiveConnection, so the ViewModel can't tell them apart. provideLiveConnection(useFake = …) is the intended single switch.
