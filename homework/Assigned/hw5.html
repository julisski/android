How to work this assignment

1

Open the project folder

In Android Studio: File ▸ Open…, then select the project folder itself (e.g. Hw5ProfileCard) — not a file inside it.

2

Let Gradle sync, then Run

Wait for the Gradle sync to finish, pick an emulator, and press Run ▶. It builds and shows a partial screen — the parts already done for you, plus visible placeholders where the TODOs are.

3

Edit MainActivity.kt to complete the TODOs

Open the one file named in each project section, find each numbered // TODO, replace its placeholder with real Compose, and re-run to watch the screen fill in.

One file per project. Everything you change lives in a single MainActivity.kt. The theme wrapper (ui/theme/Theme.kt), the Scaffold, the data classes, the sample data, and the centered layout shell are already written — you only fill in the TODOs. The starter always compiles and always shows something, so you can run after every small change.
How to submit. For each project, submit (1) your finished MainActivity.kt and (2) an emulator/device screenshot of the completed screen. Tag each change with a comment naming the TODO (// TODO 1, // TODO 2, …) and comment every line you add in the same heavy teaching style as the surrounding starter code. Starter-only means: no solution is provided — the hints name the API, the rest is yours.
Stuck on a single skill? Re-open the matching lab. Every TODO below names exactly which lab(s) it builds on, so if a TODO feels unfamiliar, that lab is your worked example for the one idea you're missing.

Project 1. Hw5ProfileCard — a profile card Android Studio

A single profile card, centered on screen, assembled from several aligned pieces.

Open, run, then complete three TODOs in MainActivity.kt
Project folder Hw5ProfileCard
Package com.example.hw5profilecard
Theme wrapper Hw5ProfileCardTheme
Material 3 · Compose BOM 2026.02.01

File to edit: app/src/main/java/com/example/hw5profilecard/MainActivity.kt

What the finished screen looks like — a single profile card centered on screen:

  • a circular avatar showing the person's initials, with a small green "online" dot pinned to its bottom-right corner;
  • the person's name (bold, titleLarge) and "@handle · Role" (muted) beneath it;
  • a row of three stats — Posts / Followers / Following — that split the width evenly, each a big bold number above a small muted label;
  • a row of two equal-width buttons: a filled "Follow" and an outlined "Message".
Already done for you (do not re-implement). The Scaffold; a Profile data class (name, handle, role, posts:Int, followers:Int, following:Int); a sample val ada = Profile(...); the outer centered Column (fillMaxSize, padding, horizontalAlignment = CenterHorizontally); the avatar Box (a ~96.dp circle via Modifier.size(96.dp).clip(CircleShape).background(brandColor) with a centered initials Text); the name Text; and the "@handle · role" Text. Your job is the three TODOs below.
TODO 1 Online badge builds on Lab 10 — Box + align

Add a small (~16.dp) green dot to the bottom-end corner of the avatar Box, so it reads as an "online" indicator overlapping the avatar's edge. It must sit on top of the avatar in that one corner, not below it in the layout flow.

Hint (API only): place the dot inside the avatar Box and pin it with Modifier.align(Alignment.BottomEnd). A small Box with .size(16.dp).clip(CircleShape).background(...) makes a clean dot.
TODO 2 Stats row builds on Labs 8 & 9 — weight

Make the three stats (Posts / Followers / Following) split the row evenly: each of the three cells takes one equal share of the width, with the big bold number on top and the small muted label centered below it.

Hint (API only): give each cell Modifier.weight(1f) inside the Row so the three shares are equal; center each cell's contents with a Column + horizontalAlignment = Alignment.CenterHorizontally.
TODO 3 Action buttons builds on Labs 7 & 8 — arrangement + weight

Add a Row with a filled Button("Follow") and an OutlinedButton("Message"). Each button takes an equal half of the row width, and the two are separated by a fixed 12.dp gap.

Hint (API only): give each button Modifier.weight(1f), and set the Row's horizontalArrangement = Arrangement.spacedBy(12.dp).

Done when…

  • the green online dot sits in the avatar's bottom-right corner, overlapping its edge;
  • the three stats are exactly equal-width, each centered, number bold above a muted label;
  • "Follow" and "Message" are equal-width with a visible gap between them;
  • the app builds and runs, and no TODO placeholder text is still visible on screen.
Deliver: finished MainActivity.kt + one emulator/device screenshot of the completed card.

Project 2. Hw5SettingsList — a settings list Android Studio

A scrollable list of settings rows that all line up, with toggles that actually flip.

Open, run, then complete the TODOs in MainActivity.kt
Project folder Hw5SettingsList
Package com.example.hw5settingslist
Theme wrapper Hw5SettingsListTheme
Material 3 · Compose BOM 2026.02.01

File to edit: app/src/main/java/com/example/hw5settingslist/MainActivity.kt

What the finished screen looks like — a scrollable settings list:

  • a "Settings" screen title;
  • six rows; each row = a leading icon + a label that fills the middle + a trailing control on the right edge — a Switch for toggle settings, a "›" chevron Text for navigation settings;
  • the toggle switches actually flip when tapped;
  • a divider separating the main settings from an "About" footer row.
Already done for you (do not re-implement). The Scaffold + "Settings" title; an enum Kind { TOGGLE, NAV }; a SettingItem data class (icon: String emoji, label: String, kind: Kind, initiallyOn: Boolean = false); a val settings = listOf(...) of six items — Notifications, Dark mode, Data saver as TOGGLE; Privacy, Language, About as NAV; and the SettingRow composable signature. Your job is the TODOs below.
TODO 1 Show every row builds on Lab 1 — layout + lists

Render all items from settings — the starter only shows a placeholder / a single row. You need a scrollable container that produces one SettingRow per item.

Hint (API only): either a Column(Modifier.verticalScroll(rememberScrollState())) with settings.forEach { ... }, or a LazyColumn { items(settings) { ... } }.
TODO 2 Row layout builds on Labs 6 & 8 — Row cross-axis + weight

Implement SettingRow as a Row: the leading icon first, then the label Text, then the trailing control. The label must fill the middle so the trailing control is pushed all the way to the end of the row, and the icon / label / control must be vertically centered with each other.

Hint (API only): give the label Text a Modifier.weight(1f) (that's what pushes the control to the end), and set the Row's verticalAlignment = Alignment.CenterVertically.
TODO 3 Make toggles work builds on Lab 5 — state

Make each TOGGLE row's Switch actually flip on and off when tapped. Give each toggle row its own remembered boolean state seeded from item.initiallyOn, feed it to the switch's checked, and update it from onCheckedChange.

Hint (API only): hold the state with var on by remember { mutableStateOf(item.initiallyOn) }; the Switch takes checked = on and onCheckedChange = { on = it }. (The NAV rows show the "›" chevron instead of a switch.)
TODO 4 (optional) Divider before About structure / arrangement

Add a HorizontalDivider just above the "About" footer row, so the About item reads as a separated footer rather than just another setting in the list.

Hint (API only): emit a HorizontalDivider() before the row whose label is "About" (or before the last item).

Done when…

  • all six rows render and the list scrolls if it doesn't fit;
  • in every row the icon is on the left, the label fills the middle, and the control sits flush on the right edge, all vertically centered;
  • each toggle row's switch flips and stays flipped when you tap it; NAV rows show a "›" chevron;
  • (optional) a divider sits above the About row;
  • the app builds and runs, and no TODO placeholder text is still visible on screen.
Deliver: finished MainActivity.kt + one emulator/device screenshot showing at least one toggle flipped on.
